# v0.9.0 milestone: SBOM-aware release assurance

Scope after v0.8.0:

## SBOM and provenance

- Generate a release SBOM and publish it with release assets.
- Keep `checksums.txt` covering binaries and SBOM artifacts.
- Add HermesScan rules that detect release workflows publishing artifacts without an SBOM signal.
- Document where HermesScan fits beside SBOM, checksum, SARIF, and provenance workflows.

## Rule model

- Support conservative file-level absence checks for cases where line-oriented regex rules are not enough.
- Keep absence rules advisory unless the trigger and missing context are both specific.
- Add precision tests for every SBOM-related rule.

## Packaging

- Consider Scoop packaging first.
- Consider Homebrew packaging after the Windows packaging path is stable.

## Documentation

- Add an SBOM and release assurance guide.
- Add examples for release workflows that publish binaries, checksums, SBOMs, and scan reports together.
