# v0.7.0 milestone: Rule quality and adoption polish

## Goals

- Reduce false positives in high-noise rules.
- Improve rule documentation and remediation guidance.
- Improve adoption for existing repositories.
- Add release and binary smoke-test confidence.

## Candidate issues

- Improve fixed-port rule precision.
- Improve package-cache rule precision.
- Add generated rule reference documentation.
- Add release binary smoke tests.
- Add changed-only scan examples.
- Add false-positive issue template.
- Add baseline adoption guide.
- Add safer `pull_request_target` remediation examples.
- Add configurable severity examples per rule.

## Non-goals

- Do not become a full CI platform.
- Do not replace ShellCheck, PSScriptAnalyzer, or actionlint.
- Do not require a language runtime at scan time beyond the native HermesScan binary.
